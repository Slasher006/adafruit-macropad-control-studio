# Contributing

## Set up a development environment

```bash
python3 -m venv .venv
.venv/bin/pip install -e '.[test,docs]'
```

## Run the checks

```bash
QT_QPA_PLATFORM=offscreen .venv/bin/python -m pytest -q
.venv/bin/python -m py_compile \
  desktop/macropad_configurator/*.py \
  device/*.py \
  tools/*.py
.venv/bin/python tools/generate_configurator_tutorial.py
git diff --check
```

Device-independent tests must pass before a change is submitted. Hardware
tests write to a connected MacroPad and should be run only when the device and
target mount have been verified.

## Profile changes

The profile index is `device/profiles/index.json`. Every listed profile must
contain exactly 12 keys per layout, OLED labels no longer than six characters,
and supported action steps.

Maintained sublayouts are generated by:

```bash
.venv/bin/python tools/build_builtin_subprofiles.py
```

Run the command twice or compare the profile diff before and after running it
to confirm that generation remains deterministic.

## Documentation

Update the GUI guide and profile catalog when behavior or built-in mappings
change. Regenerate the tutorial PDF after editing its generator:

```bash
.venv/bin/python tools/generate_configurator_tutorial.py
```

The final PDF belongs in `output/pdf/`. Temporary rendered pages used for
visual review belong under `tmp/` and must not be committed.
